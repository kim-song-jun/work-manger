## 1. 프로덕트 개요

### 미션

**"회사가 직원을 믿고, 직원이 회사를 신뢰하게 한다."**

근무 관리는 감시 도구가 아니라 *신뢰를 숫자로 증명하는* 도구다. 직원은 공정하게 기록되고, 관리자는 명확하게 확인하고, 팀은 서로 어디 있는지 부담 없이 알 수 있어야 한다.

### 핵심 기능

| # | 기능 | 설명 |
|---|---|---|
| 1 | **연차 자동 발생** | 근속 규칙(법정 15일 + 근속연차)에 따라 매년 1월 1일 자동 발생. 관리자 개입 불필요. |
| 2 | **소멸 자동 안내** | 남은 연차 소멸 90일/30일/7일 전 푸시 · 이메일 · 대시보드 배지로 자동 안내. |
| 3 | **고정 출퇴근 + 초과근무 승인** | 정규 시간(09:00–18:00) 외 타각은 자동으로 **초과근무 요청 시트**로 전환. 상급자 승인 전까지는 "대기" 상태. |
| 4 | **위치 기반 체크인** | GPS + Wi-Fi SSID 조합으로 "본사 / 재택 / 외근" 자동 판별. 사용자는 버튼만 한 번 누르면 됨. |
| 5 | **한글/영문 동등 지원** | 모든 화면 i18n. 두 언어를 1급 시민으로 대우 — 폰트 스택, 줄높이, 구두점까지 각각 최적화. |
| 6 | **전 직원 근무 상태** | 모든 직원이 실시간으로 서로의 근무 위치/상태를 부담 없이 볼 수 있음 (동의 기반). |

### 플랫폼

- **모바일 앱** (iOS/Android) — 주 플랫폼. 출퇴근, 연차 신청, 팀 확인.
- **웹 (개인)** — 상세 기록·리포트·설정 조회.
- **웹 (관리자)** — 승인, 연차 관리, 리포트, 직원 관리.
- **데스크탑 앱** — macOS 메뉴바 / Windows 시스템 트레이에 상주. 자동 체크인, 근무 시간 실시간 표시.
- **홈화면 위젯** — iOS/Android 홈에서 바로 확인.

## 2. 디자인 원칙

Toss의 디자인 철학을 차용하되, 근태 도메인에 맞게 재해석했다.

### 01. 시간에 여백을 (Breathing Room for Time)

시간과 연차는 이 프로덕트의 "돈"이다. 숫자가 중요한 정보는 일반 텍스트보다 **최소 1.5배의 공간**을 준다. 공간이 신뢰를 만든다.

```
❌ 나쁜 예: "오늘 근무 7h 42m (출근 08:54)"
✓ 좋은 예:
  오늘 근무
  7h 42m       ← 30px / 700 / tabular-nums
  출근 08:54    ← 13px / 400 / grey-500
```

### 02. 점진적 밀도 (Progressive Density)

- **요약**은 넓게 (홈 화면, 대시보드 상단)
- **상세**는 촘촘하게 (기록 조회, 관리자 테이블)

사용자가 깊이 들어갈수록 정보 밀도는 높아져도 된다. 하지만 첫 화면은 언제나 숨쉴 공간이 있어야 한다.

### 03. 한 화면, 한 행동 (One Screen, One Action)

Primary 버튼은 화면당 **하나**. 두 개가 필요하면 화면을 둘로 나눈다. 사용자가 "다음에 뭐 할지"를 한 번에 알 수 있어야 한다.

### 04. 블루는 인터랙션만 (Blue = Interactive)

`#3182F6`(Toss Blue)은 **탭할 수 있는 곳에만** 쓴다. 장식·그래프 배경·포인트 색으로는 절대 사용하지 않는다. 다른 색이 필요하면 grey-900(Dark), success, warn을 쓴다.

### 05. 절제가 곧 신뢰 (Restraint as Trust)

- 그림자: 단일 레이어, 검정 낮은 투명도 (`rgba(0,0,0,0.04)` ~ `rgba(0,0,0,0.12)`)
- 애니메이션: 대부분 `250ms` · 성공 확인에만 `spring`
- 이모지: 브랜드/로고·특별 이벤트 외 사용 금지
- 화려함은 비용이다 — 피로감, 주의 분산, 로딩 부담.

### 06. 한글과 영문은 동급 (Both Scripts First-Class)

`lang=ko`일 때 `Pretendard`, `lang=en`일 때 같은 Toss Product Sans 라틴 서브셋. 줄높이도 각각 최적화 (한글 150%, 영문 140%). Fallback 스택 모두 고려.

### 07. 숫자는 타이포그래피 (Numbers as Typography)

근무 시간, 연차, 금액 등 숫자는 `font-weight: 700 + font-variant-numeric: tabular-nums`로 고정. 본문 굵기(400)와 **절대 섞지 않는다**. 숫자 전용 스케일: XL(40) / LG(30) / MD(22).

### 08. 여백은 자산 (Whitespace is Content)

정보를 더 담는 대신 **다음 화면으로 나눈다**. 모바일은 한 화면당 3-4개의 주요 섹션만. "조금 비어 보여도" OK.

## 3. 디자인 토큰

### 3.1 토큰 체계

```
─ Primitive ────────── Semantic ───────── Component
blue-500: #3182F6  →  --brand          →  .hf-btn-primary { background: var(--brand) }
grey-900: #191F28  →  --text-primary   →  .hf-field-label { color: var(--text-primary) }
```

Primitive(원시값) 위에 Semantic(의미) 위에 Component(컴포넌트)가 쌓이는 3계층 구조. 테마 전환(light/dark, mint/violet/coral)은 Primitive는 두고 Semantic만 바꾼다.

### 3.2 Spacing

4px 그리드 · 8의 배수 선호.

| 토큰 | 값 | 용도 |
|---|---|---|
| `--sp-1` | 4px | 아이콘/텍스트 간격 |
| `--sp-2` | 8px | 인풋 내부, 칩 패딩 |
| `--sp-3` | 12px | 카드 내부, 리스트 간격 |
| `--sp-4` | 16px | 섹션 패딩 (모바일) |
| `--sp-5` | 20px | 카드 외곽 |
| `--sp-6` | 24px | 섹션 간격 (모바일) |
| `--sp-8` | 32px | 섹션 간격 (웹) |
| `--sp-12` | 48px | 주요 블록 간격 (웹) |

### 3.3 Radius

| 토큰 | 값 | 용도 |
|---|---|---|
| `--r-xs` | 6px | 태그, 인풋 |
| `--r-sm` | 10px | 버튼 medium/large |
| `--r-md` | 14px | 카드, 시트 |
| `--r-lg` | 20px | 큰 카드, 모달 |
| `--r-pill` | 999px | 칩, 스위치, dot |

모서리는 **크게** 가는 편이다 — Toss 특유의 말랑한 느낌.

### 3.4 Shadow

| 토큰 | 값 | 용도 |
|---|---|---|
| `--shadow-1` | `0 1px 2px rgba(0,0,0,0.04)` | 기본 카드 |
| `--shadow-2` | `0 2px 8px rgba(0,0,0,0.06)` | 호버 상태 |
| `--shadow-3` | `0 8px 24px rgba(0,0,0,0.08)` | 떠있는 시트, 토스트 |
| `--shadow-4` | `0 20px 64px rgba(0,0,0,0.12)` | 모달, 포커스 |

**단일 레이어 · 검정 · 낮은 투명도**. 컬러 그림자 금지.

## 4. 타이포그래피

### 4.1 폰트 스택

```css
--font-kr:  'Toss Product Sans', 'Pretendard', -apple-system, 
            'Apple SD Gothic Neo', 'SF Pro Display', Roboto, 
            'Noto Sans KR', sans-serif;

--font-num: 'Toss Product Sans', 'Pretendard', 'SF Pro Display',
            -apple-system, system-ui, sans-serif;
```

- **Toss Product Sans**: 한/영/숫자 모두 같은 굵기 시각 밀도로 설계 → 첫 선택
- **Pretendard**: 웹 fallback (오픈소스, 한글 최적화)
- **시스템 폰트**: 최종 fallback

### 4.2 타입 스케일

9단계 + 숫자 전용 3단계.

| 클래스 | 크기 | 굵기 | 행간 | 용도 |
|---|---|---|---|---|
| `.t-display-hero` | 30 | 700 | 40 | 홈 인사말 ("오늘도 기록해볼까요?") |
| `.t-display-lg` | 26 | 700 | 36 | 섹션 타이틀, 연차 잔여 |
| `.t-heading-lg` | 22 | 700 | 30 | 승인 건수, 작은 카운터 |
| `.t-heading` | 20 | 600 | 28 | 카드 타이틀 |
| `.t-subtitle` | 16 | 600 | 24 | 리스트 아이템 제목 |
| `.t-body-lg` | 16 | 400 | 24 | 본문 (중요) |
| `.t-body` | 14 | 400 | 22 | 본문 (기본) |
| `.t-body-strong` | 14 | 600 | 22 | 강조된 본문 |
| `.t-body-sm` | 13 | 400 | 20 | 서브텍스트, 메타 |
| `.t-caption` | 12 | 400 | 18 | 캡션, 레이블 |
| `.t-caption-strong` | 12 | 600 | 18 | 상태 라벨, 카테고리 |

### 4.3 숫자 전용 스케일

**tabular-nums + 700 + tight letter-spacing**. 시간과 돈은 흐트러지지 않아야 한다.

| 클래스 | 크기 | 용도 |
|---|---|---|
| `.t-number-xl` | 40 | "11일" (연차 대시보드 히어로) |
| `.t-number-lg` | 30 | "7h 42m" (오늘 근무 시간) |
| `.t-number-md` | 22 | "09:00" (출퇴근 시각) |

### 4.4 굵기 규칙

**세 가지만 쓴다.**
- **400**: 본문
- **600**: 강조 (버튼, 레이블, 카드 제목)
- **700**: 헤딩, 숫자

500/800은 시각적 노이즈를 만들어 금지.

## 6. 모션 & 애니메이션

### 6.1 Duration 토큰

| 토큰 | 값 | 용도 |
|---|---|---|
| `--motion-instant` | 0ms | 토글 플립 (애니메이션 없음) |
| `--motion-fast` | 150ms | Hover, Press |
| `--motion-standard` | 250ms | **기본** — 시트, 탭, 카드 전환 |
| `--motion-slow` | 400ms | 성공 체크마크 |
| `--motion-page` | 350ms | 페이지 전환 |

### 6.2 Easing 토큰

| 토큰 | 값 | 용도 |
|---|---|---|
| `--ease-enter` | `cubic-bezier(0, 0, 0.2, 1)` | 나타나기 |
| `--ease-exit` | `cubic-bezier(0.4, 0, 1, 1)` | 사라지기 |
| `--ease-standard` | `cubic-bezier(0.4, 0, 0.2, 1)` | 양방향 기본 |
| `--ease-spring` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | **한 번만** — 성공 체크 |

### 6.3 Signature Motions

**1. Number Slide-in (Money-moves)**

근무 시간, 연차 숫자가 바뀔 때 20px 아래에서 페이드 + 슬라이드 업.
```css
.anim-num-in { animation: num-in 250ms var(--ease-enter) both; }
```

**2. Check-mark Draw**

연차 승인, 출근 완료 등에서만 1회 사용. Spring easing으로 살짝 튀어오름.
```css
.anim-check path { 
  stroke-dasharray: 40;
  animation: check-draw 400ms var(--ease-spring) forwards;
}
```

**3. Sheet Slide-up**

바텀 시트, 모달. 40px 아래에서 페이드 + 슬라이드 업.
```css
.anim-sheet { animation: sheet-up 250ms var(--ease-enter) both; }
```

**4. Skeleton Shimmer**

로딩 시 1.2초 linear loop. 좌→우 shine.

### 6.4 Reduced Motion

`@media (prefers-reduced-motion: reduce)`에서 모든 motion 토큰을 `0ms`로 collapse. 애니메이션 자체를 끄지 않고 duration만 0으로 → 상태는 바뀌되 움직임만 없음.

## 7. 컴포넌트

### 7.1 버튼

#### Variant (6종)

| 클래스 | 배경 | 텍스트 | 용도 |
|---|---|---|---|

## 8. 상태 디자인 (State Design)

한 화면은 실제로 **4-5개 상태**를 가진다. 모두 명시적으로 디자인한다.

### 8.1 Empty

첫 사용자, 기록 없음. 

```
아이콘 (22px, grey-400)
   ↓
"아직 신청한 연차가 없어요."    ← t-subtitle
   ↓
한 문장 설명 (optional)        ← t-body-sm, grey-500
   ↓
[연차 신청하기]                 ← 약한 CTA (weak variant)
```

**원칙**: 사용자 탓 금지. "당신이 등록하지 않아서" 같은 문구 X. 중립적인 과거형 서술.

### 8.2 Loading

- **Skeleton**: 레이아웃 위치 고정이 중요한 곳 (카드 리스트)
- **Dash (`—`)**: 숫자/금액/시간
- **Spinner**: 다른 방법 없을 때만 (체크인 GPS 대기)

**절대 금지**: 로딩 중 layout shift. 최종 컨텐츠와 동일한 크기로 placeholder.

### 8.3 Error

#### 인라인 (필드 에러)
```
❌ 회사 이메일이 아니에요. 도메인을 확인해주세요.
```
- `--danger` 보더 + 한 문장
- **Blameless + Specific + Actionable** 3박자
- "잘못 입력됐습니다" (blamy, vague) → "회사 이메일이 아니에요" (blameless, specific)

#### Toast (페이지 에러)
```
[ ⚠ GPS를 잠시 읽지 못했어요. Wi-Fi로 대신 확인해드릴게요. ]
```
- grey-900 배경 + 흰 텍스트
- 3초 자동 사라짐
- 사용자 액션 있으면 [다시 시도] 추가

### 8.4 Success

**화면 전체 전용 Success 스크린**을 쓴다. 작은 토스트로 때우지 않는다.

```
   (원형 success-soft 배경)
        ✓  ← anim-check spring
   
   "연차 신청이 접수됐어요"    ← t-display-lg
   
   박민지 PM이 확인한 뒤 알려드릴게요.  ← t-body-lg
   
   ┌─────────────────────────┐
   │  7월 15-17일             │   
   │  3일 · 반차 없음          │
   │  승인자 박민지            │
   └─────────────────────────┘
   
   [ 홈으로 ]
```

연차 신청, 퇴근 완료 같은 중요 액션은 반드시 이 패턴.

### 8.5 Partial

데이터 일부만 로드됨. 나머지는 `—` + 재시도.

### 9.1 모바일 · 홈 (출퇴근)

**파일**: `hifi/mobile.jsx` → `MobHomeSlide / MobHomeTap / MobHomeCards`

홈 화면은 사용자가 하루에 최소 2번 여는 화면. 3가지 UX 방향을 **"Tweaks에서 선택"** 방식으로 제공한다.

#### A. 슬라이드 (MobHomeSlide) — 기본값
```
상단  인사 + 날짜 + 위치 자동 감지 (LocBadge: "본사 · Wi-Fi")
중앙  [     →  밀어서 출근     ] (56px 버튼, 밀면 체크인)
하단  오늘 근무 7h 42m · 정규 09:00 도착 / 18:00 퇴근 예정
```
**왜 슬라이드?** 실수 방지 + 만족감. 탭 한 번보다 인지적 확인이 강하다.

#### B. 탭 (MobHomeTap) — 미니멀
큰 원형 버튼 (직경 180px) 중앙에 "출근" 하나. 누르면 `pulse + check-draw` 애니메이션.  
**왜 탭?** 속도 우선. 출근길 지하철에서 흔들리는 환경에 유리.

#### C. 카드 스택 (MobHomeCards) — 정보형
오늘 근무, 이번 주 요약, 팀 3명, 연차 잔여를 카드 4장으로 보여주고 하단 Fab으로 출근.  
**왜 카드?** 출퇴근 외 다른 정보까지 한 화면에서 확인하고 싶은 사용자용.

#### 공통 요소
- **LocBadge**: 상단 우측, 위치 자동 감지 결과 ("본사 · Wi-Fi 감지" / "재택 · GPS 확인" / "외근")
- **BottomTabs**: 홈 / 팀 / 연차 / 마이 4탭
- **Money-moves**: 근무 시간 숫자는 1분 단위로 `anim-num-in`으로 업데이트

#### 상태
- **Empty (첫 사용자)**: "첫 출근을 환영해요 🙂 근처에서 Wi-Fi를 찾아볼게요..."
- **Off-hours**: 정규 시간 18:00 이후 탭 시 → 초과근무 시트로 전환 (→ 9.2)
- **Offline**: "연결이 불안정해요. 잠시 후 다시 시도해주세요."

### 9.2 모바일 · 초과근무

**파일**: `hifi/mobile.jsx` → `MobOvertime`

정규 시간 외 체크아웃을 시도하면 **자동으로 바텀 시트**가 올라온다.

```
┌───────────────────────────────────┐
│      ═                            │  ← drag handle
│                                   │
│  정규 시간이 지났어요              │  ← t-display-lg
│  초과근무로 기록할까요?            │  ← t-body-lg
│                                   │
│  예상 초과  2h 15m                 │  ← t-number-md, warn
│  승인자     박민지 PM               │
│                                   │
│  사유 (선택)                       │
│  ┌─────────────────────────┐      │
│  │ _________________________│      │
│  └─────────────────────────┘      │
│                                   │
│  [ 나중에 기록하기 ]               │  ← ghost
│  [ 초과근무로 요청하기 ]           │  ← primary big
└───────────────────────────────────┘
```

#### 핵심 디자인 결정
- **"초과근무"를 숨기지 않는다**. 자동으로 기록하지 않고, 사용자가 명시적으로 요청하게 한다. 노동법 투명성.
- **승인자 이름을 미리 보여준다**. 누구에게 가는지 불분명하면 사용자는 "승인 안 해주면 어쩌지" 걱정한다.
- **사유는 선택**. 강제하면 거짓 사유를 만든다.

- 각 직원이 "내 상태 공유 OFF" 토글 가능 (설정에서). 기본은 ON.
- 휴게 상태는 자동 감지 대신 본인이 버튼을 눌러야 함.

## 10. UI 커스터마이즈 시스템

사용자는 앱 UI를 **6가지 축**으로 커스터마이즈할 수 있다. 이는 9.5.2 "화면 꾸미기" 페이지에서, Tweaks 토글에서는 제작/리뷰용으로.

| 축 | 옵션 | 영향 |
|---|---|---|
| 언어 | 한국어 / English | 전체 텍스트 + 폰트 스택 |
| 테마 | 자동 / 라이트 / 다크 | body 클래스 → CSS 변수 재지정 |
| 브랜드 | Blue(기본) / Mint / Violet / Coral | `--brand` 계열만 교체 |
| 폰트 크기 | sm(14) / md(15) / lg(17) | body font-size → rem 기반 전부 연쇄 |
| 홈 레이아웃 | A 슬라이드 / B 탭 / C 카드 | 모바일 홈 컴포넌트 교체 |
| 팀 뷰 | 그리드 / 팀별 / 타임라인 | 모바일 팀 화면 컴포넌트 교체 |

**기술적 구현**:
- `body.theme-dark`, `body.theme-mint`, `body.font-lg` 등 클래스 기반
- `TWEAKS` 상태를 localStorage + 서버 동기화
- EditMode에서는 `postMessage`로 호스트에 edits 전송

## 11. 한글/영문 i18n

### 원칙

1. **두 언어 동등**. 한국어가 "원본"이 아니라 둘 다 원본.
2. **폰트 스택 각각**: `lang=ko`는 Pretendard, `lang=en`은 Toss Product Sans 라틴.
3. **줄높이 분리**: 한글 150%, 영문 140% (CSS에서 자동)
4. **구두점 localize**: "09:00–18:00" (한) / "9:00 AM – 6:00 PM" (영)
5. **숫자 단위**: "11일" / "11 days" (11 d는 금지 — 축약은 가독성 훼손)

### 구현

`hifi/i18n.jsx`에 `T_KR / T_EN` 두 테이블. `useT()` hook이 현재 언어에 맞는 번역 반환.

```jsx
const t = useT();
<h1>{t('welcome_back')}</h1>  
// ko: "오늘도 기록해볼까요?"  
// en: "Welcome back — ready to log in?"
```

### 주의

- **영어 번역은 영문 UX 원어민 리뷰 필수**. 직역 금지.
- **한글 줄바꿈**: `word-break: keep-all` + `overflow-wrap: break-word`
- **영문 대문자**: 헤드라인만 첫글자 대문자 (sentence case). ALL CAPS 금지 (공격적 느낌).

## 12. 접근성

### 12.1 WCAG 2.2 AA 준수

- 텍스트 대비: 모든 body 이상 AA (4.5:1 이상)
- 인터랙티브 요소 대비: AA (3:1 이상)
- 대비 검증은 `--grey-600 이상만 본문 사용`으로 토큰 수준에서 강제

### 12.2 터치 타겟

- 모바일 최소 **44×44pt** (iOS HIG) / **48×48dp** (Material)
- 버튼 medium(40)은 웹 전용. 모바일은 항상 large(48) 이상.

### 12.3 키보드 네비게이션

- Tab 순서 논리적
- Focus ring: `--brand` 2px outline + 2px offset
- Esc로 모달/시트 닫기

### 12.4 스크린 리더

- 아이콘 버튼 전부 `aria-label`
- 상태 변경 시 `aria-live="polite"` (출근 완료, 승인됨 등)
- 숫자 읽기: "7h 42m" → "7 hours 42 minutes"로 `aria-label` 제공

### 12.5 모션 민감성

- `prefers-reduced-motion: reduce` 시 모든 duration 0ms
- 특히 parallax, 자동 재생 animation 금지

### 12.6 색상 독립성

- 상태를 **색상으로만** 구분하지 않는다.
- Dot + 라벨 + 아이콘 3중 표기.
- 색맹 사용자 테스트 (프로타노피아/듀테라노피아 필터)

## 13. 다크 모드

### 13.1 전략

**Primitive 자체를 재정의**하지 않고 **Semantic만 교체**.

```css
body.theme-dark {
  --brand: #4D8DF8;           /* 약간 밝게 */
  --brand-soft: #1A2F4E;       /* 대비 유지 */
  --grey-900: #F9FAFB;         /* 반전 */
  --grey-100: #242B35;         /* 반전 */
  --success-soft: #0E2E23;     /* 어두운 tint */
  /* ... */
}
```

### 13.2 원칙

1. **완전한 반전 금지**. 완전한 검정(#000)은 피로. `#131720`을 기본 배경으로.
2. **채도 낮춤**. 다크 모드에서 채도 100% 컬러는 튀어 보임. success, warn 모두 살짝 desaturate.
3. **그림자 대신 보더**. 다크 모드에서 그림자는 거의 안 보임. `border: 1px solid --grey-200` (이 때 grey-200은 어두운 환경 보더값)

### 13.3 전환

- 사용자 선택: 자동(system) / 라이트 / 다크
- 전환 애니메이션: 350ms fade (harsh flash 방지)

## 14. 디자인 결정 기록 (ADR)

중요한 결정과 그 이유를 기록. 나중에 "왜 이렇게 했지?" 싶을 때 참조.

### ADR-001 · 이모지 금지 (브랜드 외)

**결정**: 모든 화면에서 이모지 사용 금지. 로고/브랜드 일러스트 외.  
**이유**: 이모지는 플랫폼마다 렌더가 달라 디자인이 깨짐. 근태 도구는 "귀여움"보다 "신뢰"가 중요.  
**예외**: 사용자가 입력한 사유 텍스트 안의 이모지는 허용.

### ADR-002 · 정확한 GPS 좌표 비공유

**결정**: 팀원 서로에게 근무 상태(본사/재택/연차)만 공유. 정확한 위치는 **서버에도 저장 안 함**.  
**이유**: 감시 도구가 아닌 신뢰 도구. GPS 원본은 "본사 맞음/아님" 판정 후 버림.

### ADR-003 · 초과근무 자동 기록 금지

**결정**: 정규 시간 외 접속은 자동 기록하지 않고, 항상 사용자가 명시적으로 "초과근무 요청" 액션을 해야 함.  
**이유**: 노동법 투명성 + "원치 않는 야근 누적" 방지. 사용자가 자기 시간을 통제한다는 느낌.

### ADR-004 · 연차 소멸 안내 3단계

**결정**: 90일 / 30일 / 7일 전 자동 안내.  
**이유**: 90일 = 계획 가능, 30일 = 마지막 기회, 7일 = 긴급. 한 번만 하면 잊음, 너무 잦으면 무시됨.

### ADR-005 · Primary 버튼 1개 원칙

**결정**: 한 화면당 Primary 버튼 하나. 두 개 필요하면 화면 분리 또는 secondary로.  
**이유**: 선택지 과다 = 결정 마비. "지금 해야 할 한 가지"를 명확히.

### ADR-006 · 숫자 스켈레톤 대신 Dash

**결정**: 숫자 로딩 시 skeleton bar 대신 `—` (em dash) 표시.  
**이유**: Skeleton이 숫자로 갑자기 바뀌면 시선이 튐. Dash는 "지금은 모름"을 명확히 전달하고 안정적.

### ADR-007 · Toss 디자인 시스템 선택

**결정**: zero-from-scratch가 아닌 Toss 기반.  
**이유**: (1) 한국 사용자 친숙, (2) 한글 타이포그래피 최적화, (3) 개발자 에코시스템 (Pretendard), (4) 숫자 중심 UI(금융) 철학이 근태 도메인과 정합.

### ADR-008 · 3개 플랫폼 동시 디자인

**결정**: 모바일 / 웹 / 데스크탑 3개를 동시에.  
**이유**: 근태 입력은 출퇴근 중(모바일) + 업무 중(데스크탑 자동) + 리포트(웹). 하나만 하면 사용자가 다른 채널로 이탈.
