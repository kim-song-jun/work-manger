# 기능 명세서 (Feature Specification)

> **Document version**: 0.1
> **Last updated**: 2026-05-04
> **Owner**: Product

---

## 0. 개요

근무 관리 시스템의 기능 요구사항을 도메인별로 정리한다. 각 기능은 다음을 포함한다.

- **목적**: 사용자가 얻는 가치
- **사용자 플로우**: 진입점 → 액션 → 결과
- **비즈니스 룰**: 자동화 / 검증 / 권한
- **수용 기준**: 구현 완료를 판정하는 기준 (Acceptance Criteria)

---

## 1. 사용자 역할 (Roles)

| 역할 | 설명 | 주요 권한 |
|---|---|---|
| `EMPLOYEE` | 일반 직원 | 출퇴근 기록, 본인 연차/초과근무 신청, 팀원 상태 조회 |
| `MANAGER` | 팀 리더 / 부서장 | 팀원 신청 승인, 팀 리포트 조회 |
| `ADMIN` | HR / 운영 관리자 | 전사 직원 관리, 연차 정책, 컴플라이언스 모니터링 |
| `OWNER` | 최고 관리자 | 회사 설정, 빌링, 권한 위임 |

권한은 누적된다. `OWNER ⊃ ADMIN ⊃ MANAGER ⊃ EMPLOYEE`

## 2. 인증 & 온보딩

### 2.1 회원가입 / 로그인

- **이메일 + 비밀번호** 또는 **OAuth (Google, Kakao)** 로 가입
- 이메일 인증 필수 (가입 후 24시간 이내 미인증 시 잠금)
- 비밀번호 정책: 8자 이상, 영문 + 숫자 + 특수문자 1개 이상
- 비밀번호 분실: 이메일 링크 (15분 유효)

### 2.2 8단계 온보딩

1. **환영** — 서비스 소개
2. **회사 코드 입력** — 6자리 영숫자 코드로 회사에 합류 (관리자가 발급)
3. **프로필** — 이름, 부서, 직급, 사번
4. **위치 등록** — 본사 위치 자동 또는 수동 등록
5. **근무시간 설정** — 출근 시각 / 퇴근 시각 / 휴게시간
6. **알림 설정** — 푸시 / 이메일 / 슬랙 (선택)
7. **위젯 추천** — iOS / Android / 데스크탑 위젯 안내
8. **완료** — 첫 출근 안내

### 2.3 비즈니스 룰

- 회사 코드는 관리자 페이지에서 발급 / 회수 가능. 만료 기한 설정 가능.
- 동일 이메일로 다중 회사 가입 불가 (v1). 멀티 테넌트는 v2에서 검토.

## 3. 출퇴근 (Attendance)

### 3.1 출근 (Clock-in)

**목적**: 마찰 없이 정확한 출근 시각을 기록한다.

**플로우**:
1. 앱 진입 → GPS 위치 자동 측정
2. 등록된 본사 / 재택 위치 반경 100m 이내 → "출근하기" 버튼 활성화
3. 사용자가 슬라이드(slide-to-clock-in) 또는 큰 원형 버튼 탭
4. 서버에 `clock_in` 이벤트 기록 → 화면 상태 → 근무 중

**비즈니스 룰**:
- 출근 시각이 정규 시작시간 + 10분을 초과하면 `late` 플래그
- 위치 검증 실패 시 (GPS 차단, 위치 외) → "수동 출근 요청" 가능 (관리자 승인 필요)
- 1일 1회만 출근 가능. 재출근은 퇴근 후 가능 (예: 외근)

### 3.2 퇴근 (Clock-out)

- 출근과 동일한 UX (슬라이드 / 탭)
- 정규 종료시간 이전 퇴근 시 사유 입력 (선택)
- 정규 종료시간 이후도 같은 방식. **초과 근무는 별도 승인 플로우** (§4)

### 3.3 휴게 (Break)

- 출근 중에만 토글 가능. 누적 휴게 시간 표시.
- 점심시간(12:00~13:00 등 회사 설정)은 자동 휴게로 처리 옵션.

### 3.4 위치 기반 자동 감지

- iOS: Significant Location Change + Geofence
- Android: Geofencing API + Foreground Service
- 자동 감지 시에도 **사용자 확인** 후 기록 (자동 기록 OFF 옵션)

### 3.5 수용 기준

- [ ] 정규 출근 → 1초 이내 서버 기록 완료
- [ ] 위치 정확도 100m 이내일 때만 자동 활성화
- [ ] 오프라인 상태 출근 시 로컬 큐에 저장 → 온라인 복귀 시 동기화
- [ ] 동시 다중 디바이스 출근 시도 → 첫 요청만 성공, 나머지는 에러 반환

## 4. 초과근무 (Overtime)

### 4.1 자동 승인 요청

**룰**: 정규 종료시간 이후 출근 상태 유지 시 자동으로 상급자에게 승인 요청 발송.

- 정규 종료시간 + 10분 → 사용자에게 "오늘 초과근무 하나요?" 푸시
- 사용자가 "네" 응답 → 사유 입력 → 상급자 승인 요청 생성
- 사용자가 "아니오" 응답 또는 무응답 10분 → 자동 퇴근 처리

### 4.2 수동 신청

- 사용자가 직접 초과근무 신청 (예정된 야근)
- 시작 시각 / 종료 예정 시각 / 사유 입력

### 4.3 승인

- 상급자가 인박스 또는 푸시에서 한 번 탭으로 승인 / 반려 가능
- 자동 승인 정책 설정 가능 (관리자): 주 X시간 이내는 자동 승인 등

### 4.4 수용 기준

- [ ] 자동 요청은 사용자 응답 후에만 생성 (스팸 방지)
- [ ] 승인 / 반려 시 신청자에게 즉시 알림
- [ ] 승인된 초과근무 시간은 근태 리포트에 합산

## 5. 연차 (Annual Leave)

### 5.1 자동 발생

**룰** (한국 근로기준법 기준 · 회사별 커스터마이즈 가능):
- 입사 1년 미만: 매월 만근 시 1일씩 부여 (최대 11일)
- 입사 1년 이상: 매년 1월 1일 (또는 회계연도 기준일) 15일 부여
- 3년차부터 2년마다 1일 추가 (최대 25일)

### 5.2 자동 소멸 안내

- 연차 사용 만료일 30일 / 14일 / 7일 / 1일 전 자동 안내 (푸시 + 이메일)
- 만료 7일 전 풀스크린 알림 (앱 진입 시)
- 사용 촉진 제도(근로기준법 §61) 자동화 옵션 — 관리자가 활성화 시

### 5.3 신청 플로우

1. 캘린더에서 시작일 / 종료일 선택 (반차 / 종일)
2. 사유 입력 (선택, 회사 설정에 따라 필수)
3. 미리보기 → 제출
4. 상급자 승인 후 확정. 거절 시 사유 첨부.

### 5.4 팀 연차 캘린더

- 한 달 / 한 분기 단위 매트릭스 뷰
- 같은 날 연차 인원 표시, 임계치(예: 팀의 30%) 초과 시 경고

### 5.5 수용 기준

- [ ] 매일 자정에 연차 발생 / 소멸 배치 실행 (Celery beat)
- [ ] 신청 후 5분 내 상급자 알림
- [ ] 연차 잔여 = 부여 - 사용 - 소멸 (실시간 계산, 캐시)

## 6. 팀 근무 상태 (Team Status)

### 6.1 상태 종류

| 상태 | 색상 | 설명 |
|---|---|---|
| 본사 | 초록 (`--success`) | 본사 출근 |
| 재택 | 파랑 (`--brand`) | 재택근무 출근 |
| 연차 | 주황 (`--warn`) | 연차 사용 중 |
| 휴게 | 노랑 (`--caution`) | 휴게 중 |
| 퇴근 | 회색 (`--grey-400`) | 미출근 / 퇴근 완료 |

### 6.2 뷰

- **그리드**: 아바타 + 상태 점. 검색 / 부서 필터.
- **팀별 그룹**: 부서별 묶기. 출근율 / 평균 출근 시각.
- **실시간 타임라인**: 시간축에 출근 이벤트 펼치기.

### 6.3 비즈니스 룰

- 본인이 속한 회사 직원만 조회 가능
- 관리자는 전사 조회. 일반 직원은 같은 부서 + 협업 부서만 조회 (회사 설정)
- 개인이 "상태 비공개" 토글 시 일반 직원에게는 표시 안 됨 (관리자에게만 노출)

## 7. 관리자 (Admin)

### 7.1 대시보드

- 오늘 출근률 / 미출근 인원 / 승인 대기 건수 / 진행 중 초과근무

### 7.2 승인 통합

- 연차 / 초과근무 / 수동 출근 신청을 한 곳에서 처리
- 일괄 승인, 사유 일괄 입력

### 7.3 직원 관리

- 직원 목록 / 검색 / CSV 일괄 등록
- 권한 변경, 부서 이동, 비활성화
- 직원 상세 — 출근 이력, 연차 잔여 / 사용, 초과근무 누적

### 7.4 연차 정책

- 회사 회계연도 기준일 설정
- 부여 규칙 커스터마이즈 (입사 X년 → Y일)
- 일괄 부여 / 회수 (특별 휴가 등)

### 7.5 리포트

- 월간 / 주간 근태 리포트 (직원별 / 부서별)
- 출근율, 평균 출근 시각, 초과근무 시간, 연차 사용률
- CSV / PDF 내보내기

### 7.6 컴플라이언스 (52시간)

- 주 52시간 초과 자동 경고
- 초과 시 출근 차단 옵션 (회사 설정)
- 관리자 전용 컴플라이언스 보드

## 8. 알림 (Notifications)

| 트리거 | 채널 | 수신자 |
|---|---|---|
| 출근 시각 임박 | 푸시 | 본인 |
| 초과근무 자동 요청 | 푸시 + 이메일 | 본인 → 상급자 |
| 승인 / 반려 | 푸시 + 인박스 | 신청자 |
| 연차 소멸 임박 | 푸시 + 이메일 | 본인 |
| 팀원 신청 대기 | 푸시 + 인박스 | 상급자 |
| 컴플라이언스 경고 | 푸시 + 이메일 | 관리자 |

수신자가 알림 채널별 ON/OFF 설정 가능 (마이 → 알림).

## 9. 다국어 (i18n)

- 지원 언어: 한국어 (`ko`), 영어 (`en`)
- 디바이스 OS 언어 자동 감지, 사용자 명시 설정이 우선
- 모든 사용자 노출 문자열은 키 기반. 하드코딩 금지.
- 날짜 / 시각 / 숫자는 `Intl.DateTimeFormat`, `Intl.NumberFormat` 사용

---

## 10. 접근성 (a11y)

- WCAG 2.1 AA 준수
- 폰트 크기 3단계 (Sm / Md / Lg) — 사용자 설정
- 색약 대응: 상태 표시는 색상 + 아이콘 / 라벨 병기
- 키보드 전체 조작 가능 (웹 / 데스크탑)
- `prefers-reduced-motion` 시 모션 비활성화

---

## 11. 비범위 (Out of Scope · v1)

다음은 v1 범위에 포함하지 않는다.

- 급여 계산 / 4대보험 연동
- 복지포인트 / 비용 청구
- 외부 캘린더 양방향 동기화 (Google / Outlook)
- 다중 법인 / 멀티 테넌트
- SAML / SCIM SSO

v1.x 이후 우선순위는 [`roadmap.md`](../roadmap.md) 참조.
