# 소유주 가이드

소유주는 관리자 권한 + **회사 설정 / 데이터 주권** 권한.

## 회사 설정 (`/admin/settings`)

| 섹션 | 내용 |
|---|---|
| 회사 정보 | 회사명, 사업자번호, 대표 이메일, 주소 |
| 브랜드 | 로고 (PNG), 브랜드 컬러 (#hex), 로그인 화면 배경 |
| 출근 정책 | 본사 / 지점 / 재택 활성화, 지오펜스 반경 |
| 알림 정책 | 매니저 알림 / 직원 알림 / 마케팅 알림 |
| 보안 | 2FA 강제, 로그인 lockout 임계, 비밀번호 정책 |
| 컴플라이언스 | 주 52시간 강제 차단 / 경고만 / 비활성 |
| 데이터 보존 | audit log 보존 기간 (기본 90일), 출근 기록 (3년) |

## 데이터 주권

### 회사 전체 export
- **마이** → **소유주 도구** → **데이터 export**
- 대상: 직원 명부, 출근 기록, 연차, 승인, 감사 로그, 알림 발송 이력.
- 형식: ZIP (CSV per 도메인) 또는 JSON (machine-readable).
- 처리 시간: 회사 규모에 따라 5분 ~ 1시간. 완료 시 메일 통지.

### 회사 전체 삭제 (계약 종료)
- **마이** → **소유주 도구** → **계정 폐기 요청**
- 운영팀이 14일 유예 후 처리:
  - 즉시: 로그인 차단
  - 1일차: 모든 사용자 비활성화
  - 7일차: PII 익명화
  - 14일차: 영구 삭제 (감사 로그만 7년 보존 — 법정 의무)
- [SOP — 데이터 삭제 요청](../operations/sop/sop-data-deletion-request.md) 참조.

## 사고 대응

- 보안 사고 / 데이터 유출 의심 시: 운영팀 핫라인 + [Runbook §보안 사고](../operations/runbook.md#보안-사고).
- 24시간 내 통지 의무 (한국 개인정보보호법 §34).

## 운영팀 위임

- 일상 운영은 ADMIN 권한 직원에게 위임 권장.
- 소유주는 분기별 1회 감사 로그 + 데이터 보존 정책 점검.
